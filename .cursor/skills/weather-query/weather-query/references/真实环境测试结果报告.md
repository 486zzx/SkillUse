## 三、输出结构验证

对通过用例的 JSON 输出抽查结论：

- **result.location**：含 `name`、`id`（心知城市 ID），与请求地点一致或为心知解析结果（如朝阳区→朝阳）。
- **result.current**：含 `text`、`code`、`temperature`、`feels_like` 等，与心知实况接口一致。
- **result.daily**：数组，每项含 `date`、`high`、`low`、`text_day`、`text_night` 等，符合 SKILL 约定。
- **result.suggestion**：在带 `--with-suggestion` 时存在（中国城市可有具体指数）。
- **result.air**：在带 `--with-air` 时存在。
- **失败用例（无参数）**：`success: false`，`error` 为「缺少参数：需要至少一个位置参数（地点），例如：run_weather_search.py 北京」。

## 五、建议

1. **回归**：脚本或 SKILL 输入输出约定变更后，重新执行 `run_real_env_tests.py` 做回归。
2. **Key 安全**：真实环境依赖有效 Key；建议 CI 中使用环境变量注入 Key，勿将 Key 提交仓库。
3. **扩展**：若需覆盖更多城市、更多日期或无效地点，可在 `run_real_env_tests.py` 的 `TEST_CASES` 中追加用例，并同步更新本报告与《真实环境测试用例全集》。
