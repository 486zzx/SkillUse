# Skill 层功能介绍与参数说明

本文档从 **Skill 本身** 的视角描述：当 Agent 或上层调用「使用该 Skill」时，Skill 的**功能**是什么、**参数**是什么。  
**Skill 层统一约定**：每个 Skill 只接受**一个参数**——**用户问题**（用户用自然语言表达的需求或问题）。Agent 从用户问题中解析出具体条件（如出发站、日期等），再传给脚本执行，但对外接口上 Skill 的输入仅为「用户问题」。

---

## 1. train-ticket-search（列车班次查询）

### Skill 功能

- **用途**：根据用户自然语言查询**火车票/列车班次**（高铁、动车、普速等），返回指定出发站→到达站、指定出发日期的车次列表及票价等信息。
- **能力**：单次调用只查**一对站点、一个出发日期**；支持自然语言出发时间/到达时间（如「明天」「后天下午五点后」「五点后到」），由 Skill 内部归一为日期与时间段；支持按**车型**筛选（G/D/Z/T/K/O/F/S）；支持按**价格、出发时间、到达时间、历时**排序；支持限制返回条数。
- **触发场景**：用户问火车票、高铁、动车、某地到某地火车、出发站/到达站/出发日期、最便宜、按价格/时间排序、历时等；即使用户未说「火车」但语境为铁路出行时也使用本 Skill。
- **不适用**：用户明确要查**飞机/航班**时改用航班查询 Skill。只支持15天内（包括当天）的搜索。

### Skill 参数

| 参数名 | 类型 | 描述 | 必选/可选 | 未提供时 |
|--------|------|------|------------|----------|
| 用户问题 | string | 用户用自然语言表达的需求，如「明天北京到上海的高铁，最便宜的」；Agent 从中解析出发站、到达站、出发时间、到达时间（若有）、车型、排序等并传给脚本 | **必选** | 无法调用 |

## 2. weather-query（天气查询）

### Skill 功能

- **用途**：根据用户给出的**城市/地点**查询天气：实况、逐日预报，可选生活指数与空气质量。
- **能力**：单次调用只查**一个城市/地点**；可指定起始日期（今天/明天/后天/昨天/前天或具体日期），得到从该日起的逐日预报；可开关「生活指数」（仅中国城市）、「空气质量」；支持心知多种 location 形式（中文名、拼音、城市 ID、经纬度）。
- **触发场景**：用户问某地天气、今天/明天/后天天气、气温、会不会下雨、穿衣建议、空气质量、PM2.5 等。
- **范围**：心知支持全国约 3156、全球约 24,962 个城市；**生活指数仅中国城市**。

### Skill 参数

| 参数名 | 类型 | 描述 | 必选/可选 | 未提供时 |
|--------|------|------|------------|----------|
| 用户问题 | string | 用户用自然语言表达的需求，如「北京明天天气」「上海天气要穿衣建议」；Agent 从中解析地点、日期（若有）、是否要生活指数/空气质量等并传给脚本 | **必选** | 无法调用 |

## 3. surround-search（周边搜索）

### Skill 功能

- **用途**：根据用户给出的**目标地址**，查询该地址**周边的 POI**（如餐厅、咖啡店、加油站等），返回名称、地址、距离、类型等信息。
- **能力**：单次调用只针对**一个目标地址**；先对目标地址做地理编码得到坐标，再以该点为圆心做周边搜索；支持按**关键词**筛选 POI 类型、按**距离**或**综合权重**排序、设置**半径**和**返回条数**；目标地址**必须与用户输入一致**，不得自行补全省市区等。
- **触发场景**：用户问「附近有什么」「周边」「周围」「找餐厅/咖啡/加油站」「某地周边」「按距离」等；即使用户只说「查一下 XX 附近」未提「周边搜索」也应优先用本 Skill。

### Skill 参数

| 参数名 | 类型 | 描述 | 必选/可选 | 未提供时 |
|--------|------|------|------------|----------|
| 用户问题 | string | 用户用自然语言表达的需求，如「三里屯附近有什么餐厅」「北京西站周边咖啡店」；Agent 从中解析目标地址、城市（若有）、关键词、排序、半径、条数等并传给脚本 | **必选** | 无法调用 |

---

## 4. multi-engine-search（多引擎聚合搜索）

### Skill 功能

- **用途**：对用户的**问题或搜索需求**进行多引擎并行检索（百度 + Tavily），结果经去重、排序、过滤后统一返回，供 Agent 组织回答。
- **能力**：支持仅传**用户问题**（单 query 多引擎）；或从用户问题中拆出/扩展出多组关键词，对「每个关键词 × 每个引擎」并行请求后合并；可选**查询改写**（将问题改写成更适搜的 query）；结果条数由内部 pipeline 控制（如上限 50 条）。
- **触发场景**：用户需要「搜一下」「查一下」「找资料」「多源搜索」「聚合搜索」，或问题需要查网上信息才能回答时使用本 Skill。

### Skill 参数

| 参数名 | 类型 | 描述 | 必选/可选 | 未提供时 |
|--------|------|------|------------|----------|
| 用户问题 | string | 用户用自然语言表达的问题或搜索需求，如「python是什么」「Rust 和 Go 性能对比」；Agent 将问题（及可选拆出的关键词）传给脚本，由脚本在多引擎上检索并返回统一结果 | **必选** | 无法调用 |

## 5. flight-search（国内航班查询）

### Skill 功能

- **用途**：根据用户自然语言查询**国内航班**（出发地→目的地、出发日期），返回航班列表；支持直飞或中转、按价格/起飞时间/机型等筛选与排序。
- **能力**：单次调用只查**一段行程**（一个出发地、一个目的地、一个出发日期）；出发地/目的地支持**城市名、机场名、省份**，Skill 内部解析为 IATA 三字码；日期支持自然语言（明天、后天、3月5号等）；**范围**：中国境内（含大陆与台湾）；境外不查并提示。
- **触发场景**：用户问查机票、查航班、某地到某地、出发日期、直飞/中转、按价格或时间排序、起飞时间等。
- **默认**：用户未说「可中转」时按**直飞**（maxSegments=1）查询；**仅当用户明确要求排序/筛选**时才传 sort_by、max_price 等；不支持过去时间航班查询/

### Skill 参数

| 参数名 | 类型 | 描述 | 必选/可选 | 未提供时 |
|--------|------|------|------------|----------|
| 用户问题 | string | 用户用自然语言表达的需求，如「明天上海到北京的机票」「虹桥飞成都直飞按价格排序」；Agent 从中解析出发地、目的地、日期、是否中转、筛选与排序等并传给脚本 | **必选** | 无法调用 |

---

## 汇总：Skill 功能与参数

| Skill | 功能简述 | 参数 |
|-------|----------|------|
| train-ticket-search | 列车班次查询（一对站点、单日；自然语言时间、车型、排序） | **用户问题** |
| weather-query | 天气查询（单城市；实况+逐日；可选生活指数、空气质量） | **用户问题** |
| surround-search | 周边 POI 查询（单目标地址；关键词、排序、半径、条数） | **用户问题** |
| multi-engine-search | 多引擎聚合搜索（百度+Tavily；可选关键词扩展与查询改写） | **用户问题** |
| flight-search | 国内航班查询（单段行程；直飞/中转、筛选、排序） | **用户问题** |

**说明**：Skill 层对外只接受**一个参数「用户问题」**。具体条件（出发站、日期、地点等）由 Agent 从用户问题中解析后，在调用脚本时传入，属于 Skill 内部执行约定，而非 Skill 的对外参数。
