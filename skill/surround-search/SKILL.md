# 周边搜索

根据用户自然语言请求，查询**目标地址周边的 POI 信息**（如餐厅、咖啡店、加油站等）：由模型提取关键词后，调用本 skill 提供的脚本完成「地理编码 → 周边搜索 → 结果整合」，输出统一格式的周边信息供你组织回复。

**推荐**：直接调用 **`scripts/run_surround_search.py`**，传入**位置参数 + 可选长参数**；脚本内部完成参数转化、高德 API 调用与结果整合。拿到返回的 `pois` 后，由你根据上下文以自然语言呈现给用户（表格或列表均可），**须完整展示返回的 POI 列表**，不得折叠或只显示前几条。

**重要**：传给脚本的**目标地址必须与用户输入一致**。不要补充用户没说的省、市、区、街道、门牌号，不要自行「纠正」或「补全」地址，否则可能查错地点（例如用户只说「万达广场」未提城市，若你自行补成「某市万达广场」可能查到错误城市的结果）。

---

## 1. 输入与输出

- **输入**：用户问题中应包含「目标地址」；可选包含**搜索关键词**（如餐厅、咖啡店、加油站）、排序方式（按距离/综合）、半径或条数。
- **目标地址**：**必须原样使用用户输入的地址，不得补充、纠正或改写。** 不要写入用户没提到的省、市、区、街道、门牌号等任何信息；否则可能张冠李戴（例如用户只说「软件园」「万达广场」等，多地都有，若自行补成某省某市会查错地点）。若用户给出的是结构化地址（省+市+区+街道+门牌号），就原样传；若用户只说了简称或地标名，就只传该简称或地标名，并在回复中可提示「若您能提供更完整的地址（省+市+区+街道+门牌号），检索会更准确」。
- **输出**：脚本返回 `{ success, pois, total_count, query_summary?, error? }`。`pois` 为数组，每项含 `name`、`address`、`distance`（米）、`poi_type` 等。你据此整理成可读回复。
- **失败时**：`success === false` 时根据 `error` 向用户说明原因（如未找到目标地址、缺少 Key 等）。

---

## 2. 你的职责（模型侧）

1. **提取关键词**  
   从用户输入中识别：  
   - **目标地址**（必填）：**仅传递用户原话，不要自行补充或纠正。** 用户说了「北京市朝阳区三里屯路」就传该句；用户只说「万达广场」或「国贸」就只传「万达广场」或「国贸」——不要添加用户未提供的省、市、区、街道等，否则易导致检索到错误地点。可在回复中建议用户「提供更完整地址（省+市+区+街道+门牌号）可提高准确率」，但调用脚本时不得写入用户没说的内容。  
   - **城市**（可选，可不填）：仅当**用户明确说出**城市名、或地址中**已明确包含**城市（如「北京市朝阳区…」）时，才传 `--city`。**若无法从用户输入中确定城市，不要猜测或推理**——不传 `--city` 即可（脚本会走全国检索），否则推断错误可能导致 API 结果有误。  
   - **搜索关键词**（可选）：如餐厅、咖啡店、加油站、酒店；对应脚本的 `--keyword` 参数。  
   - **排序**（可选）：用户说「按距离」「最近的」时传 `--sort-by distance`（默认）；「综合」时传 `--sort-by weight`。  
   - **半径**（可选）：如「方圆 3 公里」→ `--radius 3000`（米）。  
   - **条数**（可选）：仅当用户明确指定条数时传 `--max-results N`（如「只要 5 个」→ `--max-results 5`）；**用户未指定时不传**，脚本默认返回 25 条。

2. **调用脚本**  
   使用**命令行长参数**调用 `run_surround_search.py`，不要用 stdin 传 JSON。工作目录为 skill 根目录或 `scripts` 所在目录；需设置环境变量 **`AMAP_KEY`** 为高德 Web 服务 API Key。

3. **呈现结果**  
   根据返回的 `pois` 与 `query_summary` 组织回复，完整列出 POI（名称、地址、距离等），便于用户选择。

## 3. 脚本用法（run_surround_search.py）

**环境**：设置 **`AMAP_KEY`**（高德 Web 服务 API Key），否则脚本报错并返回 `success: false`。

**必填（位置参数）**：  
- 第 1 个参数：**目标地址** `location`。**必须与用户输入一致，不得自行补充省/市/区/街道/门牌号。** 用户给的是结构化地址（省+市+区+街道+门牌号）就原样传；用户只给简称或地标名就只传该简称或地标名。

**可选（--key 值）**：  
- `--city`：可选，**可不填**。仅当用户明确给出或地址中已包含城市名时填写；**不确定城市时不要推理**，不传此项由 API 全国检索，避免推断错误导致结果有误。  
- `--keyword`：**搜索关键词**（如 `餐厅`、`咖啡店`、`加油站`），脚本会按该关键词请求周边搜索；不传则使用高德默认范围。  
- `--sort-by`：`distance`（按距离，默认）或 `weight`（综合）。  
- `--radius`：搜索半径，单位米，0–50000，超过按 5000 处理。  
- `--max-results`：返回条数上限。**用户未指定时不传**，脚本默认 25 条；用户指定时传，建议 ≤25（高德单页限制）。

**示例**

```bash
# 用户明确说了「北京市西城区莲花池东路」时，原样传该地址；--keyword 传搜索关键词
python scripts/run_surround_search.py 北京市西城区莲花池东路118号 --city 北京 --keyword 餐厅 --max-results 10

# 用户只说「三里屯」「国贸」等时，只传用户原话，不补省市区；--keyword 传 咖啡店/餐厅
python scripts/run_surround_search.py 三里屯 --keyword 咖啡店 --radius 2000
python scripts/run_surround_search.py 国贸 --keyword 餐厅

# 用户只说「万达广场」等未带城市时，只传该句，不要自行补成「某市万达广场」
python scripts/run_surround_search.py 万达广场 --keyword 餐厅
```

**输出结构**（成功时）：  
`success: true`，`pois`: `[{ name, address, distance, poi_type, location? }]`，`total_count`，`query_summary`: `{ location, city?, keyword?, sort_by?, radius? }`。  
失败时：`success: false`，`error`: 原因说明。

## 5. 流程小结

1. 用户输入 → 你提取：**目标地址**（必填，**原样传递用户所说，不补充、不纠正**）、城市/**搜索关键词**/排序/半径/条数（可选）。**城市**仅当用户明确给出或地址中已含时才传，不确定时**不推理、不传**。**搜索关键词**通过 `--keyword` 传入。  
2. 设置 `AMAP_KEY`，执行 `run_surround_search.py <用户原话地址> [--city ...] [--keyword <关键词>] [--sort-by distance|weight] [--radius N] [--max-results N]`。  
3. 根据返回的 `pois` 与 `query_summary` 组织回复，**完整展示** POI 列表。

脚本内部会：将地址（+ 城市）请求高德地理编码 → 取第一个结果的坐标 → 用该坐标按**关键词**请求周边搜索（半径、排序、条数）→ 整理为统一 POI 列表并输出。
