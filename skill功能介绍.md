# Skill 功能介绍与参数说明

本文档描述各 Skill 的**完整功能**与**参数**（类型、必选/可选、范围/格式、默认值）。调用方式均为命令行（工作目录为对应 skill 的 `scripts` 或 skill 根目录）。

---

## 1. train-ticket-search（列车班次查询）

### 功能说明

- **一次查询**：根据「出发站、到达站、出发时间」查询**单日**列车班次；每次调用仅查**一对站点、一个出发日期**。
- **可查数据**：车次号、出发站/到达站、出发/到达时刻、历时、席别与票价（如商务座、一等座、二等座等）、是否可预订等；数据来自聚合数据「列车站到站时刻表」API，仅支持**出发日期 15 天内**。
- **能力**：支持出发站/到达站用**站点名称**（如北京、上海）或**站点编码**（如 VNP、OHH）；出发时间、到达时间支持**自然语言**（如「明天」「后天下午五点后」「五点后」），由脚本用 jionlp 归一为单日日期 + 时间段；支持按**车型**筛选（G/D/Z/T/K/O/F/S）；支持按**出发时段**筛选（凌晨/上午/下午/晚上）；支持按**到达时间范围**筛选（如「五点后」「晚上前」）；支持对结果**排序**（价格、出发时间、到达时间、历时，升序/降序）；支持限制返回条数。

### 入口脚本

`run_train_search.py`

### 参数表


| 参数名              | 类型             | 描述                            | 必选/可选  | 范围/格式                                                                                                                    | 未指定时默认         |
| ---------------- | -------------- | ----------------------------- | ------ | ------------------------------------------------------------------------------------------------------------------------ | -------------- |
| `from_station`   | string（位置参数 1） | 出发地名称                         | **必选** | 如：北京、VNP                                                                                                                 | -              |
| `to_station`     | string（位置参数 2） | 到达站名称或站点编码                    | **必选** | 如：上海、OHH                                                                                                                 | -              |
| `departure_time` | string（位置参数 3） | 出发时间（自然语言或日期）                 | **必选** | 如：明天、后天下午五点后、2025-12-21                                                                                                  | -              |
| `--arrival-time` | string         | 到达时间范围（自然语言），用于过滤到达时间在该范围内的车次 | 可选     | 如：五点后、晚上前                                                                                                                | 不传则不按到达时间过滤    |
| `--train-type`   | string（可多次）    | 车型筛选；每次一个值                    | 可选     | 枚举：`G`(高铁/城际)、`D`(动车)、`Z`(直达特快)、`T`(特快)、`K`(快速)、`O`(其他)、`F`(复兴号)、`S`(智能动车组)                                              | 不传则不过滤车型       |
| `--sort-by`      | string         | 排序方式                          | 可选     | 枚举：`price_asc`、`price_desc`、`departure_asc`、`departure_desc`、`arrival_asc`、`arrival_desc`、`duration_asc`、`duration_desc` | 不传则保持 API 返回顺序 |
| `--max-results`  | int            | 最多返回的班次条数                     | 可选     | 正整数                                                                                                                      | 不传则返回全部        |


### 环境变量

- `JUHE_TRAIN_API_KEY`：聚合数据接口 key，**必配**（否则脚本报错）。可在 skill 根目录建 `.env` 配置，evals 的 conftest 会加载。
- `JUHE_TRAIN_DEBUG=1`：在 stderr 打印请求 URL（key 脱敏）与接口返回摘要，便于排查。

## 2. weather-query（天气查询）

### 功能说明

- **一次查询**：每次调用仅查**一个城市/地点**的天气；可指定**一个日期**（今天/明天/后天或自然语言），得到以该日为起点的预报。
- **可查数据**：  
  - **实况**：当前天气、气温、体感、湿度、风向风速、能见度等。  
  - **逐日预报**：从指定日起连续 N 天的日间/夜间天气、气温范围、降水等。  
  - **生活指数**（可选）：需传 `--with-suggestion`，仅中国城市有效，如穿衣、运动、洗车等建议。  
  - **空气质量**（可选）：需传 `--with-air`，如 PM2.5、AQI 等。
- **数据源**：心知天气 API v3（实况、逐日、生活指数、空气质量）；认证为私钥 key。

### 入口脚本

`run_weather_search.py`（路径：`weather-query/scripts/`）

### 参数表


| 参数名                 | 类型             | 描述                | 必选/可选  | 范围/格式                          | 未指定时默认    |
| ------------------- | -------------- | ----------------- | ------ | ------------------------------ | --------- |
| `location`          | string（位置参数 1） | 查询地点/城市           | **必选** | 中文或英文，如：北京、上海、beijing          | -         |
| `date`              | string（位置参数 2） | 预报起始日期（自然语言或日期）   | 可选     | 如：今天、明天、后天、2025-12-21；不传表示从今天起 | 不传则从「今天」起 |
| `--days`            | int            | 逐日预报天数            | 可选     | 1～15                           | 3         |
| `--with-suggestion` | flag           | 是否拉取生活指数（仅中国城市有效） | 可选     | 无参数                            | 不传则不拉取    |
| `--with-air`        | flag           | 是否拉取空气质量          | 可选     | 无参数                            | 不传则不拉取    |
| `--language`        | string         | 返回语言              | 可选     | 如 zh-Hans、en 等                 | zh-Hans   |
| `--unit`            | string         | 温度单位              | 可选     | c / f 等                        | c         |


### 环境变量

- `SENIVERSE_KEY`：心知私钥；**必选其一**（与下方签名二选一）。未设置时脚本内可能有默认 key（仅示例，建议自建）。
- `SENIVERSE_UID` + `SENIVERSE_PRIVATE_KEY`：心知公钥+私钥（签名验证）；与 `SENIVERSE_KEY` **二选一**。

## 3. surround-search（周边搜索）

### 功能说明

- **一次查询**：根据一个**目标地址**（如「北京西站」「三里屯」）先做**地理编码**得到坐标，再以该点为圆心做**周边 POI 搜索**；每次调用只针对**一个目标地址**。
- **可查数据**：周边 POI 的名称、地址、距离（米）、类型、经纬度；支持按**关键词**筛选（如餐厅、咖啡店、加油站），支持按**距离**或**权重**排序，支持设置**搜索半径**和**返回条数上限**。
- **数据源**：高德 Web 服务（地理编码 + 周边搜索）。

### 入口脚本

`run_surround_search.py`

### 参数表


| 参数名             | 类型           | 描述                 | 必选/可选  | 范围/格式                            | 未指定时默认     |
| --------------- | ------------ | ------------------ | ------ | -------------------------------- | ---------- |
| `location`      | string（位置参数） | 目标地址，用于地理编码与周边搜索圆心 | **必选** | 如：北京西站、三里屯                       | -          |
| `--city`        | string       | 地理编码时的城市限定，提高解析准确度 | 可选     | 城市名，如：北京                         | 不传则由高德自动推断 |
| `--keyword`     | string       | 周边 POI 关键词筛选       | 可选     | 如：餐厅、咖啡店、加油站                     | 不传则不过滤类型   |
| `--sort-by`     | string       | 结果排序规则             | 可选     | 枚举：`distance`（按距离）、`weight`（按权重） | distance   |
| `--radius`      | int          | 搜索半径（米）            | 可选     | 0～50000                          | 5000       |
| `--max-results` | int          | 返回条数上限             | 可选     | 正整数；高德单页限制，建议 ≤25                | 25         |


### 环境变量

- `AMAP_KEY`：高德 Web 服务 API Key；**必配**，否则脚本返回 `success: false` 并提示未配置。

## 4. multi-engine-search（多引擎聚合搜索）

### 功能说明

- **一次调用**：根据用户问题（和可选的多组关键词）进行**多引擎并行搜索**，结果**去重、排序、合并**后输出；每次调用完成一次「问句 + 多关键词 × 多引擎」的聚合。
- **可查数据**：来自百度千帆搜索与 Tavily 的网页结果（标题、链接、摘要等）；支持仅传**用户问题**（单 query 多引擎），或**用户问题 + 多个 -k 关键词**（每个关键词 × 每个引擎并行请求，再合并）。
- **能力**：可选**查询改写**（将用户问题改写成更适合搜索的 query）；结果经 pipeline 过滤与条数上限控制；需配置百度与 Tavily 的 API Key。

### 入口脚本

`aggregate_search.py`

### 参数表


| 参数名                 | 类型           | 描述                    | 必选/可选  | 范围/格式                        | 未指定时默认                 |
| ------------------- | ------------ | --------------------- | ------ | ---------------------------- | ---------------------- |
| `query`             | string（位置参数） | 用户问题/搜索问句             | **必选** | 任意字符串                        | 可与 `--stdin` 配合从标准输入读  |
| `-k` / `--keywords` | string（可多次）  | 每条为一条搜索词；关键词 × 引擎并行请求 | 可选     | 如：-k "python" -k "python 教程" | 不传则仅用 query（可改写后）单条多引擎 |


说明：未提供 query 且未提供任何 `-k` 时，脚本输出错误 JSON 并退出。内部 pipeline 的 `max_items` 固定为 50（不通过命令行暴露）。

### 环境变量

- `TAVILY_API_KEY`：Tavily API Key；**必配**，否则 Tavily 引擎返回「未配置」类错误。
- `BAIDU_APPBUILDER_API_KEY`：百度千帆/AppBuilder API Key；**必配**，否则百度引擎返回「未配置」类错误。
- `AGGREGATE_QUERY_REWRITE`：设为 `0`、`false`、`no`、`off`、`disabled` 可关闭查询改写；默认启用。
- `AGGREGATE_TIMING=1`：在 stderr 输出各阶段耗时，便于排查性能。

## 5. flight-search（航班查询）

### 功能说明

- **一次查询**：每次调用仅查**一段行程**（一个出发地 → 一个目的地、一个出发日期）；支持**直飞或中转**（由 `maxSegments` 控制）。
- **可查数据**：航班列表（航班号、起飞/到达时刻、日期、历时、机型、票价等）；出发地/目的地支持**城市名、机场名、省份**，脚本会解析为 IATA 三字码再请求聚合数据航班 API；日期支持**自然语言**（如明天、后天），由脚本归一为 yyyy-mm-dd。
- **能力**：可选按**最高票价**、**起飞时段**（最早/最晚起飞时间）、**机型**筛选；可选按**价格**、**起飞时间**、**历时**排序；航段数默认**直飞**（maxSegments=1），用户明确说可中转时可传 2/3/0。

### 入口脚本

`run_flight_search.py`

### 参数表


| 参数名                    | 类型             | 描述                        | 必选/可选  | 范围/格式                                                                       | 未指定时默认         |
| ---------------------- | -------------- | ------------------------- | ------ | --------------------------------------------------------------------------- | -------------- |
| `origin`               | string（位置参数 1） | 出发地（城市/机场/省份）             | **必选** | 如：上海、浦东、SHA                                                                 | -              |
| `destination`          | string（位置参数 2） | 目的地（城市/机场/省份）             | **必选** | 如：北京、首都、PEK                                                                 | -              |
| `date`                 | string（位置参数 3） | 出发日期（自然语言的日期）             | **必选** | 如：明天、下周一                                                                    | -              |
| `--max-segments`       | string         | 最大航段数；1=直飞，2/3/0=允许中转     | 可选     | 1、2、3、0（0 表示不限制）                                                            | 1（直飞）          |
| `--max-price`          | number         | 最高票价筛选（元）                 | 可选     | 正数                                                                          | 不传则不过滤价格       |
| `--sort-by`            | string         | 结果排序                      | 可选     | 枚举：`price_asc`、`price_desc`、`departure_asc`、`departure_desc`、`duration_asc` | 不传则保持 API 返回顺序 |
| `--min-departure-time` | string         | 最早起飞时间，过滤早于该时刻的航班         | 可选     | 格式 HH:MM，如 08:00                                                            | 不传则不过滤         |
| `--max-departure-time` | string         | 最晚起飞时间，过滤晚于该时刻的航班         | 可选     | 格式 HH:MM，如 22:00                                                            | 不传则不过滤         |
| `--equipment-contains` | string         | 机型包含关键词，只保留机型字符串包含该关键词的航班 | 可选     | 如：738、787                                                                   | 不传则不过滤机型       |


### 环境变量

- `JUHE_FLIGHT_API_KEY`：聚合数据航班 API Key；**必配**，否则接口返回错误；代码内占位为 `your-key`，需替换为真实 key。
